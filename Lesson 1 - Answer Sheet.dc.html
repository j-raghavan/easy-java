Programming Basics: Answer Key

📌 For the student: Try all three tasks yourself in the live editor (or IntelliJ) before reading this. Compare your working code and output to the solutions below. If yours differs, look for why — that's the real lesson.

1 Warm-up — use a variable

Create an int called score set to 90, and print "My score is 90" using the variable.

int score = 90; System.out.println("My score is " + score);
Expected outputMy score is 90
Key idea: the + here joins the text "My score is " with the value inside score. Because one side is a String, Java converts the number to text and glues them together (concatenation).

Common mistake: typing "My score is 90" as one string. That works visually but ignores the goal — the task is to use the variable, so if score changes, the message updates automatically.

2 Math — mixing double and int

A ticket is $12.50; you buy 3. Use a double for price, an int for quantity, then print the total.

double price = 12.50; int quantity = 3; double total = price * quantity; System.out.println("Total cost: $" + total);
Expected outputTotal cost: $37.5
Key idea: when you multiply a double (12.5) by an int (3), Java promotes the result to a double — so total must be a double, not an int. Java prints 37.5, not 37.50 (trailing zeros are dropped).

Common mistake: declaring int total. That causes a compile error (possible lossy conversion from double to int) because a decimal can't fit in an int box.

3 Loop — print 1 through 5

Use a for loop to print the numbers 1 through 5, each on its own line.

for (int i = 1; i <= 5; i++) { System.out.println(i); }
Expected output1 2 3 4 5
Reading the loop: start i at 1 → check i <= 5 → if true, run the body → then i++ adds 1 → check again. It stops when i becomes 6 because 6 <= 5 is false.

Common mistakes: using i < 5 (stops at 4) or starting i = 0 (prints 0–4). Off-by-one errors are the #1 loop bug — always trace the first and last pass by hand.
