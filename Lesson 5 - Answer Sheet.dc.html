Objects & Recursion: Answer Key

📌 For the student: Task 1 is class design (structure matters more than output). Tasks 2 & 3 are recursion — the key is spotting the base case. Attempt all before reading.

1 Design a Student class

Two fields (name, int grade), a constructor, and a promote() method.

public class Student { String name; // field int grade; // field public Student(String n, int g) { // constructor name = n; grade = g; } public void promote() { // behavior grade = grade + 1; } }
What to check: the two fields are declared at the top, the constructor's parameters are copied into the fields, and promote() changes the object's own grade field. Creating one: Student s = new Student("Kai", 10); s.promote(); → Kai's grade becomes 11.

Common mistake: naming the constructor something other than Student, or giving it a return type. A constructor has no return type and must match the class name exactly.

2 Recursive countdown

countdown(int n) prints n → 1 then "Blastoff!".

void countdown(int n) { if (n <= 0) { // base case: stop System.out.println("Blastoff!"); return; } System.out.println(n); // do this level's work countdown(n - 1); // recurse on a smaller n } countdown(5);
Expected output5 4 3 2 1 Blastoff!
Key idea: the base case (n <= 0) stops the recursion. Each call prints its own n, then calls itself with n - 1 — a smaller problem that marches toward the base case.

3 Recursive sum

int sumTo(int n) returns 1 + 2 + … + n.

int sumTo(int n) { if (n <= 1) { // base case return 1; } return n + sumTo(n - 1); // this n, plus the sum of the rest } System.out.println(sumTo(10));
Expected output55
Key idea: sumTo(10) = 10 + sumTo(9) = 10 + 9 + sumTo(8) … down to sumTo(1) = 1. Adding 1+2+…+10 gives 55. The recursive call must move toward the base case (n - 1), or it never stops.

Common mistake: forgetting the base case, or recursing with the same n. Either way the method calls itself forever and throws StackOverflowError.
