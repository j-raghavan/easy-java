Concept 02 · The pipeline

Java's journey: source → bytecode → running ⚙️

Java isn't read directly by the chip. It takes a clever two-step trip:

→
📦

Bytecode — half-way language

☕
JVM
Runs bytecode on any device

🌍 Write once, run anywhere

The same .class bytecode runs on Windows, Mac, a phone, or a spacecraft — as long as that device has a JVM. That portability is why Java is everywhere.

🐞 The compiler is your friend

javac checks your grammar first. If you forget a semicolon, it refuses to compile and tells you the line — before the program ever runs.

Concept 05

Two kinds of errors 🚦

Knowing when an error strikes tells you where to look for the fix.

🛑 Compile-time error

Caught by javac before the program runs. Usually a grammar/spelling mistake. The program never starts.

System.out.println("hi") ← missing ;

💥 Runtime error

Code is grammatically fine, so it compiles — but something goes wrong while running, and it crashes.

int x = 5 / 0; ← divide by zero 💥
Rule of thumb → Red squiggles in IntelliJ before you run = compile-time. Program starts then throws an Exception = runtime.

Your assignment 📝

Use the live editor above (or IntelliJ). Attempt each yourself, then open the answer sheet to compare.

1
Memory math. A program stores 3 int variables. Each int uses 4 bytes. Create the variables and print the total bytes used (should be 12).
2
Integer division & remainder. You have 30 students and want teams of 4. Use / to print how many full teams, and % (modulo) to print how many students are left over.
3

Why it matters → AP CS A constantly asks you to predict output of integer division and modulo, and to identify error types. These three tasks drill exactly that.
