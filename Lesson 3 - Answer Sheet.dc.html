📌 For the student: Try all three yourself first. Task 3 is the classic integer-division trap — if your average printed as a whole number, that's the lesson.

1 Even or odd

Print whether an int is even or odd using % and if/else.

int n = 7; if (n % 2 == 0) { System.out.println(n + " is even"); } else { System.out.println(n + " is odd"); }
Expected output7 is odd
Key idea: n % 2 is the remainder after dividing by 2. Even numbers divide evenly, so the remainder is 0; odd numbers leave 1. The test n % 2 == 0 is the standard even check.

Common mistake: writing n % 2 = 0 (single =). One = assigns; you need == to compare — a compile-time error here.

2 Letter grade

Print A/B/C/F from an int score using if / else if / else.

int score = 85; if (score >= 90) { System.out.println("A"); } else if (score >= 80) { System.out.println("B"); } else if (score >= 70) { System.out.println("C"); } else { System.out.println("F"); }
Expected output (score = 85)B
Key idea: order matters. Java checks top to bottom and runs the first true branch, then skips the rest. Because 85 fails >= 90 but passes >= 80, it prints B and never checks the others.

Common mistake: writing separate ifs instead of else if. With separate ifs, a score of 95 would match ≥90, ≥80, and ≥70 and print three letters.

3 Average with a decimal

Average 88, 92, 79 and print it as a decimal.

int a = 88, b = 92, c = 79; int sum = a + b + c; // 259 double average = sum / 3.0; // divide by 3.0 (a double!) System.out.println("Average: " + average);
Expected outputAverage: 86.33333333333333
Key idea: the sum is 259. Writing sum / 3 (both ints) gives 86 — the decimal is lost. Dividing by 3.0 (a double) forces double division and keeps 86.333…. You could also write (double) sum / 3.

Common mistake: double average = sum / 3; — this still computes 259 / 3 = 86 as ints first, then stores 86.0. The cast/decimal must be in the division itself.
