Concept 01

The anatomy of a Java program 🦴

This skeleton never changes. Memorize its shape — you'll type it hundreds of times.

Greeter.java
public class Greeter { // the class — the container public static void main(String[] args) { // where the program starts String name = "Ada"; // a statement — ends with ; System.out.println("Hello, " + name); } }
class Greeter — the container; its name must match the file Greeter.java.
main — the starting line. The JVM always runs main first.
{ } — curly braces group code into blocks. Every { needs a matching }.
; — a semicolon ends every statement, like a period ends a sentence.

The int-vs-double trap ⚠️

Same numbers, different types, different answers. Press Step to watch:

Division.java
int intResult;intResult = 7 / 2;double dblResult;dblResult = 7.0 / 2;
intResult

The rule → If both operands are int, you get int division (decimal dropped). Make at least one a double — e.g. (double) a / b — to keep the decimal.

Making decisions: if / else if / else 🔀

Java checks each condition top to bottom and runs the first one that's true. Press Step:
