Concept 01

A class is a blueprint 📐

The class describes what every object of that type has (fields) and can do (methods). One blueprint → many objects.

Dog.java
public class Dog { String name; // field (state) int age; // field (state) public Dog(String n, int a) { // constructor name = n; age = a; } public void bark() { // method (behavior) System.out.println(name + " says woof!"); } }
class Dog — the blueprint / type.
name, age — fields: the data each dog stores.
Dog(...) — the constructor: runs when you build a new dog, sets up its fields.
bark() — a method: something a dog can do.
Why objects? → Bundling data with the code that uses it keeps big programs organized. A Dog object carries its own name everywhere it goes — no giant pile of loose variables.

Concept 03

Inheritance: build on what exists 🧬

A subclass extends a superclass — it inherits all its fields and methods, then adds more. "A Puppy is a Dog."

Puppy.java
public class Puppy extends Dog { // inherits name, age, bark() public Puppy(String n) { super(n, 0); // call Dog's constructor } public void play() { // a new behavior System.out.println(name + " chases its tail!"); } }
extends Dog — Puppy is a kind of Dog.
inherited — Puppy already has name, age, and bark() for free.
super(...) — calls the parent (Dog) constructor.
play() — a brand-new ability only Puppy has.
✓A Puppy can both bark() (inherited) and play() (its own).

Recursion: a method that calls itself 🪆

Recursion solves a big problem by solving a smaller version of itself — until a base case stops it. This runs:
