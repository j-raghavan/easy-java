Mars Rover Mission: Answer Key

📌 For the student: This is one complete solution combining all three extensions. Your code may look different and still be correct — what matters is that the loop, conditionals, .equals(), the low-battery check, and the scan counter all work.

✦ Complete rover with all 3 extensions

int battery = 100; int position = 0; int scanCount = 0; // Task 3: counter String[] commands = {"FORWARD", "SCAN", "TURN", "SCAN", "FORWARD", "CHARGE"}; for (int i = 0; i < commands.length; i++) { String cmd = commands[i]; if (cmd.equals("FORWARD")) { position = position + 10; battery = battery - 5; System.out.println("Move -> " + position + "m (battery " + battery + "%)"); } else if (cmd.equals("SCAN")) { battery = battery - 10; scanCount = scanCount + 1; // Task 3 System.out.println("Scanning rocks... (battery " + battery + "%)"); } else if (cmd.equals("CHARGE")) { battery = battery + 20; System.out.println("Solar charging... (battery " + battery + "%)"); } else if (cmd.equals("TURN")) { // Task 1: new command battery = battery - 2; System.out.println("Rotating... (battery " + battery + "%)"); } if (battery < 30) { // Task 2: safety check System.out.println(" WARNING: LOW BATTERY"); } } System.out.println("=== Mission done. Distance: " + position + "m, battery: " + battery + "%, scans: " + scanCount + " ===");
Expected outputMove -> 10m (battery 95%) Scanning rocks... (battery 85%) Rotating... (battery 83%) Scanning rocks... (battery 73%) Move -> 20m (battery 68%) Solar charging... (battery 88%) === Mission done. Distance: 20m, battery: 88%, scans: 2 ===

1 Task 1 — the TURN command

What to check: a new else if (cmd.equals("TURN")) branch that subtracts 2 from battery and prints a rotate message. It must be part of the same if/else if chain so exactly one branch runs per command.

2 Task 2 — low-battery safety check

What to check: an if (battery < 30) placed inside the loop (after the command handling) so it's checked every step. In this run the battery never drops below 30, so the warning doesn't fire — that's correct. Have students test it with a longer mission (e.g. many SCANs) to see the warning appear.

3 Task 3 — counting scans

What to check: a scanCount variable declared before the loop, incremented only inside the SCAN branch, and printed in the final summary. Here 2 SCAN commands ran → scans: 2. This is the "counter" pattern from Lesson 4.

Common mistakes: using == instead of .equals() for the commands (Lesson 4 rule); declaring scanCount inside the loop so it resets to 0 every time; or placing the low-battery check outside the loop so it only runs once at the end.
